まとめ

今週のプロダクト視点の主役は「Claude Code のデスクトップ強化」と「アーティファクトの"配れるアプリ"化」。デスクトップ版にアプリ内ブラウザが載り、Claude がドキュメントやデザインを自分で開いて操作できるようになった。アーティファクトは公開共有・複数人での同時編集・MCP コネクタ接続に対応し、Claude Code や Claude Tag から「作って配れる小さなアプリ」を出せる素材に変わりつつある。CLI は 2.1.206〜2.1.211 を一週間で連発し、Bedrock/Vertex での auto mode 既定 ON、スクリーンリーダー対応、subagent 出力の stream-json 転送を追加した。外では SpaceXAI が Grok Build（ハーネス）をオープンソース化し、Devin Fusion が Fable 5 を採用して「高いモデルなのにタスク単価は下がる」という意外なコスト観測を報告している。

Claude Code

デスクトップにアプリ内ブラウザ——Claude が docs もデザインも自分で開いて触る

Claude Code のデスクトップ版にアプリ内ブラウザが載った。ローカルの dev サーバに対してやっているのと同じ要領で、Claude がドキュメントやデザイン、任意のサイトを開いて読み、クリックし、操作できる。サンドボックス化されており、どこまで触らせるかは設定できる。

これまで「URL を貼って要約させる」止まりだった外部情報の取り込みが、エージェント自身がブラウザを運転して調べ物や動作確認まで回す形に近づいた。実装中に仕様ページを開いて確認する、デザインを見て寄せる、といった往復がセッション内で完結する。

  • @ClaudeDevs — Claude Code desktop に in-app browser

アーティファクトが"配れるアプリ"に——公開共有・同時編集・MCP コネクタ接続

アーティファクト周りが一週間で大きく前に出た。まず公開共有とマルチプレイヤー編集に対応し、リンクを知っていれば誰でも閲覧でき、複数人が同じアーティファクトを一緒に編集できるようになった（公開共有と社内共有、Team/Enterprise でのマルチプレイヤー）。しかもこれらは Claude Code のセッションからも、Slack 上の Claude Tag からも作れる。「Slack のスレッドでダッシュボードを頼むと動くページが返ってきて、そのまま社内に配れる」という導線だ。

さらに 7/15、アーティファクトが MCP コネクタを呼べるようになった。作った本人がセッションを回し直さなくても、閲覧者それぞれの MCP コネクタ経由でライブデータを取りに行き、各自が権限を持つ範囲だけを表示する。thomas（trq212）は「Claude Tag で作ったプロジェクトのダッシュボードを、他の人や自分のローカル Claude Code セッションからも編集する」使い方を挙げている。プロトタイプを作って社外にまで出せる、一度作れば MCP でライブに更新される——アーティファクトが使い捨ての生成物から、共有して使い続けるツールへと性格を変えつつある。

  • @ClaudeDevs — Artifacts が公開共有・マルチプレイヤー編集に対応
  • @ClaudeDevs — Artifacts が MCP コネクタを呼べるように
  • @trq212 — Claude Code セッションから編集するダッシュボードの例

2.1.206〜2.1.211——auto mode 既定 ON（Bedrock/Vertex）、スクリーンリーダー対応、subagent 出力の転送

この一週間で 2.1.206 から 2.1.211 まで一気に進んだ。機能面の目玉は3つ。(1) 2.1.207 で Bedrock / Vertex AI / Foundry では auto mode がオプトイン不要の既定 ON になった（disableAutoMode で無効化可、既定モデルも Opus 4.8 に）。(2) 2.1.208 でスクリーンリーダーモード（--ax-screen-reader / CLAUDE_AX_SCREEN_READER=1）が入り、プレーンテキスト描画でアクセシビリティを改善。長時間セッションのメモリリーク修正や入力応答性の改善も同時に入った。(3) 2.1.211 で --forward-subagent-text フラグ／環境変数が追加され、subagent のテキストと thinking を stream-json 出力に含められるようになった——下流のシステムがサブエージェントの中身を受け取れる。

安全側の作り込みも続く。2.1.210 では isolation: 'worktree' のサブエージェントが main リポジトリに対して git 破壊コマンドを走らせられてしまう問題や、ultracode キーワードが webhook や PR コメントなど非人間入力で発火する問題を修正。2.1.211 では権限プレビューが bidi・ゼロ幅文字を無害化し、承認メッセージの見た目を細工されないようにした。派手な新機能というより、エージェントに任せたときの土台を毎日削り込んでいる。

  • @ClaudeCodeLog — 2.1.207（Bedrock/Vertex で auto mode 既定 ON）
  • @ClaudeCodeLog — 2.1.211（subagent 出力の stream-json 転送）

「自動化してドメイン知識をインフラに焼く」——中の人とエンジニアの共通テーマ

今週は Claude Code の作り手側から、使い方の指針が相次いだ。bcherny は「昔から一番強いエンジニアは自分の作業を自動化することに時間を使ってきた（vim の自動化、lint ルールで繰り返しミスを潰す等）」とし、エージェント時代にその重要性がさらに増すと述べる。Boris の発言も oikon48 が紹介しており、「自動化し、ドメイン知識をインフラとしてエンコードすることがますます重要になる。モデルが賢くなりハーネスが成熟するほど、このエンコード作業は楽になる」という趣旨だ。

プロンプトの型についても、thomas（trq212）が「thin prompts / thick artifacts + context / thin skills（薄いプロンプト・厚い成果物とコンテキスト・薄いスキル）」という整理を投げている。ツールの新機能を追うだけでなく、繰り返す判断を skill やインフラに落とし込む——今週の Claude Code 界隈の関心は、この「エンコード」に寄っていた。

  • @bcherny — 作業の自動化の重要性について
  • @oikon48 — Boris「ドメイン知識をインフラにエンコードする」
  • @trq212 — thin prompts / thick artifacts / thin skills

ハーネスをめぐる動き

SpaceXAI が Grok Build をオープンソース化——ハーネスを公開して信頼性を上げる

SpaceXAI が Grok Build をオープンソース化し、あわせて全ユーザーの利用制限をリセットした。Grok Build CLI の Git リポジトリを含むコードを公開し、「信頼できて堅牢なハーネスを作るのを誰でも支援できるようにする」ためだとしている。前週に Grok 4.5 を「コーディング/エージェント特化」と打ち出したのに続く一手で、モデルだけでなくエージェントを動かす器（ハーネス）そのものを開くことで、外部の目とコントリビューションを取り込みにいく形だ。

プライバシー面も明示している。zero data retention（ZDR）を使うチームでは trace もコードデータも一切保持せず、API キー利用の Grok Build も ZDR を尊重するという。OpenAI の Codex が「オープンソースのハーネスを選ぶ理由」に言及していたこととも響き合い、ハーネスを開いて標準化・検証可能にする流れが各社で見えてきた。

  • @SpaceXAI — Grok Build をオープンソース化、利用制限リセット
  • @SpaceXAI — Grok Build と ZDR（データ保持）について

OpenAI、Codex アプリを ChatGPT デスクトップアプリへ統合——ChatGPT Work とともに

先週の OpenAI の大型リリースは、プロダクト／ハーネスの統合という点で product 視点でも見逃せない。5 か月前に出した Codex アプリが下地となって、新しい ChatGPT デスクトップアプリが登場し、Chat・Work・Codex が一つのアプリに同居する形になった。あわせて発表された ChatGPT Work は、Codex と GPT-5.6 を動力にアプリやファイルをまたいで行動し、必要なら数時間ひとつの仕事に張り付く新しいエージェントだ（デスクトップが最も強力、Web/モバイルにも展開）。Codex 側にも「アプリ内でファイルを直接編集」「PR を離れずに確認・レビュー」などの開発者向け強化が入った。

sama は「エージェント系プロダクト（Codex と ChatGPT Work）の利用がこの一週間で 2.5 倍」と述べている。コーディングエージェントの入り口が専用アプリから汎用のデスクトップアプリへ寄っていく動きで、Claude 側の「Chat と Cowork を一つのホームに統合」とも並走する構図になっている。

  • @OpenAI — ChatGPT Work（Codex + GPT-5.6 のエージェント）
  • @ajambrosino — 新 ChatGPT デスクトップアプリは Codex アプリが下地

ツールの実務アップデート

Devin Fusion が Fable 5 を採用——「高いモデルなのにタスク単価は下がる」

Cognition が Devin Fusion に Fable 5 を組み込んだ。興味深いのはコストの観測で、Fable 5 は Opus 4.8 より単価が高いモデルにもかかわらず、Devin Fusion 上ではタスクあたりのコストがむしろ Opus 4.8 より低くなったという。委譲（delegation）まわりでコスト効率の改善が見られたとしている。

「モデル単体の価格」ではなく「ハーネスの中で1タスクを終えるまでの総コスト」で見ると、賢いモデルほど無駄な往復が減って安く上がることがある——という現場の肌感覚を、実プロダクトの数字で裏づけた形。モデル選定を単価表だけで決めない、という判断材料になる。

  • @cognition — Devin Fusion が Fable 5 を採用（タスク単価の観測）

Codex CLI 0.144.5——危険コマンド検出を拡張、rm の強制実行をより広く捕捉

OpenAI の Codex CLI が 0.144.5 を公開。危険コマンドの検出を拡張し、強制的な rm のバリエーションをより多く捕まえ、コマンド拒否の理由をより明確に返すようにした。Claude Code 側でも未解決変数を含む rm -rf の確認やトランスクリプト改ざん防止が入っていたのと同様、エージェントにシェルを任せる前提での「事故らせない」作り込みが CLI 各社で並行して進んでいる。
