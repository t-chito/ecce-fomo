まとめ

今週のプロダクト視点の主役は「エージェントが実物を触る」方向への一段の踏み込み。Claude Code のデスクトップ版が iOS シミュレータと連携し、Claude がビルドした iOS アプリを隣のパネルで自分で起動・操作しながら仕上げられるようになった（macOS + Xcode、public beta）。セキュリティも自前のインフラに寄り、Claude Code 用の「Claude Security」プラグインが beta で登場——コミット前に手元の Claude 推論で脆弱性スキャンをかけられる。CLI は 2.1.215〜2.1.218 を一週間で連発し、`/code-review` のバックグラウンド subagent 化、ripgrep ベースの Grep 徹底、スクリーンリーダーモードの実装完了などを積んだ。外では Cognition が「Devin をどこでも動かす」Devin Outposts を出し、Codex CLI 0.145.0 がマルチエージェント V2 に踏み込んだ。そして「スキルは拾うより育てる」という現場観が、Cowork の"録画してスキル化"機能と個人エンジニアの発信の両方から浮かび上がった一週間だった。

Claude Code

デスクトップが iOS シミュレータと連携——Claude が自分でアプリを起動して仕上げる

Claude Code のデスクトップ版が iOS シミュレータと連携するようになった。iOS アプリをビルドして走らせると、会話のすぐ隣のパネルにシミュレータが開く。Claude は動いているアプリを見て、自分でタップやスクロールをして操作し、完成まで反復する。もちろんユーザーもいつでもシミュレータを自分で触れる。macOS のみ・Xcode が必要で、今日から public beta。

先週のデスクトップ版アプリ内ブラウザ（Claude が任意サイトを開いて操作する）に続く動きで、「ローカル dev サーバに対してやっていること」がモバイルの実機動作確認にまで広がった格好だ。UI を実装させたあと、シミュレータ上での挙動を Claude 自身が見て直す——という往復がセッション内で閉じる。今週の x で最大級のエンゲージメントを集めた発表で、モバイル開発をエージェントに任せる導線として注目された。

  • @ClaudeDevs — Claude Code desktop が iOS シミュレータ連携
  • @ClaudeDevs — Claude がアプリを見て操作・反復する

「Claude Security」プラグインが beta——コミット前に手元の推論で脆弱性スキャン

Claude Code 向けの Claude Security プラグインが beta で公開された。変更差分をコミット前にスキャンして脆弱性を洗い出したり、コードベース全体のフルスキャンを走らせたりが、すべてターミナルから——しかも普段使っている Claude の推論の上で——できる。外部のセキュリティ SaaS に差分を送るのではなく、いま回しているエージェント基盤の延長でセキュアレビューを済ませられるのが売りだ。

同じ週に OpenAI の Greg Brockman も「サイバー防御に自社モデルを使う codex security プラグイン」を案内しており、コーディングエージェントに脆弱性検出を組み込む動きは各社で並走している。エージェントにコードを書かせるほど、その場でのセキュリティ点検を CLI に内蔵する需要が高まっている構図だ。

  • @claudeai — Claude Security プラグイン（beta）
  • @gdb — codex security プラグイン（サイバー防御向け）

2.1.215〜2.1.218——/code-review のバックグラウンド化、ripgrep Grep、スクリーンリーダー実装完了

この一週間で 2.1.215 から 2.1.218 まで進んだ。開発体験に効く変更が中心だ。(1) 2.1.218 で /code-review がバックグラウンドの subagent として走るようになり、レビュー出力で会話が埋まらず、後続の / コマンドはそのレビューを対象に積める。(2) 2.1.217 でプロンプトが検索タスクに ripgrep ベースの Grep ツールを使うよう明示され、検索が速く正確になった。あわせて絵文字ショートコード補完（:heart: → ❤️）や、トランスクリプト書き込み失敗（ディスク満杯など）の警告も入った。(3) 先週入り始めたスクリーンリーダーモードは実装が進み、2.1.218 では単語・行削除の際に消えたテキストを読み上げるようになった。

土台の作り込みも続く。2.1.215 では Claude が /verify と /code-review を自動実行しなくなり（必要なときに手動で呼ぶ形に）、2.1.216 は長時間セッションでのメッセージ正規化が二次的に遅くなる問題を修正、sandbox.filesystem.disabled でネットワーク制御を残したままファイル分離だけ切れるようにした。2.1.217 では同時 subagent 数に既定 20 の上限を設けた。派手な新機能というより、エージェントに長く任せたときの安定性とレビュー動線を毎日削り込んでいる。

スキルは「拾う」より「育てる」

Cowork が「録画してスキル化」——作業を撮って話すと Claude が再実行できる手順に

Claude Cowork に、自分の作業を録画してスキルにする機能が入った。画面を録画しながらタスクをこなし、やっていることを声で説明すると、Claude がそれを何度でも再実行できるスキルに変換する。デスクトップアプリの「+」メニューの「Record a skill」から使え、Pro / Max / Team で利用できる。今週の x で最大のエンゲージメントを集めた発表で、「手順をコードで書き起こす」のではなく「やってみせて教える」というスキル生成の入り口が広がった。

  • @claudeai — Cowork でスキルを録画して教える

「拾ってきた Skill はほとんど役に立たない」——t_wada の運用論

この「スキルを育てる」というテーマは、個人エンジニアの発信とも響き合った。t_wada は「どこかから拾ってくる Agent Skill で役に立つものはほとんど無い（もちろん例外はある）。チームや組織が自分たちに合わせた Agent Skill を作って運用（新規作成・更新・廃棄）できる、その"スキルを育てる能力"のほうがはるかに大事」と述べる。続けて「その意味では、拾ってくる Agent Skill で最も役立つのは skill-creator（スキルを作るスキル）かもしれない」とも。

Cowork の録画スキル化が「作る」入り口を下げる一方で、価値は既製品を集めることではなく自組織の文脈に合わせて回し続けることにある——という指摘だ。ツールの新機能と現場の運用観が、今週はどちらも「スキルをどう育てるか」に収斂した。

  • @t_wada — 最も役立つのは skill-creator かも

エージェントを「どこで動かすか」

Devin Outposts——Devin を自分のマシンで動かす

Cognition が Devin Outposts を発表した。Mac mini、ラボの GPU ボックス、プライベートネットワーク内の VM、社内サービスの隣に置いた Kubernetes クラスタなど、任意のマシン上で Devin のセッションを走らせられる。Devin のエージェントループ（推論・計画）は引き続き Devin のクラウドで動き、コマンド実行・ファイル編集・リポジトリアクセスといった実作業だけを自分が管理するマシンで行う——という切り分けだ。内部サービスへのアクセスを自社のプロキシや権限の内側に閉じたまま、エージェントに手を動かさせられる。

Cloudflare Workers（エッジで隔離サンドボックス）、Daytona（スナップショットから 90ms 未満で起動）、E2B、Modal、Namespace（M5 の Mac で Xcode + computer use）、NVIDIA Brev（GPU デバッグ）と、用途別のデプロイテンプレートが同時に用意された。エージェントの「頭」はベンダー側に置きつつ「手」を自社インフラに下ろす形は、Claude Code デスクトップの iOS シミュレータ連携とも通じる、実環境で動かすための整備の流れだ。

  • @cognition — Devin Outposts 発表
  • @cognition — Cloudflare Workers での実行

Codex CLI 0.145.0——マルチエージェント V2、音声入力、スレッド履歴

OpenAI の Codex CLI が 0.145.0 を公開。目玉は3点で、(1) 音声入力とリアルタイム V3 ストリーミング、(2) マルチエージェント V2（サブエージェント、並行実行、ロール）、(3) 検索・再開・メモリ付きのスレッド履歴。Claude Code 側が subagent の背景実行や同時数上限の調整を進めているのと同じ週に、Codex もサブエージェントと並行制御を本格化させた。CLI 各社が「1人のエージェント」から「複数を束ねて回す」段階へ、ほぼ足並みを揃えて進んでいる。

現場の視点

「大げさすぎる」という直感が、エージェントのオーバーエンジニアリングを止める

コーディングエージェントを実務で回すうえでの勘所も共有された。t_wada は「『この問題の解決策がここまで大げさになるのはおかしい』という直感が、コーディングエージェントのオーバーエンジニアリング傾向へのブレーキになっている。ただ、この直感を（エージェントに）教えるのがなかなか難しい」と指摘する。エージェントは放っておくと過剰に汎用的・堅牢な実装に振れやすく、それを押し戻すのは人間側の「筋の良さ」の感覚だ——という現場感だ。

あわせて「AI と関係なくやるべきことをやる。するとそこに AI の生産性が掛け算される」とも投稿しており、土台となる設計判断や規律があってこそエージェントの生産性が乗る、という見立てを示している。ツールの新機能を追う一方で、任せる前提の"効かせ方"に関心が集まった一週間だった。

  • @t_wada — AI の生産性は掛け算
