まとめ

今週はモデル面の地殻変動が主役だった。6/30、Claude Code の既定モデルが Claude Sonnet 5 に切り替わった——1M トークンのネイティブコンテキストを備え、最上位 Opus 4.8 に迫る性能を Sonnet 価格で出すと位置づけられ、8/31 まで $2/$10 per Mtok の導入価格が付く。並行して、輸出規制の解除で Fable 5 が復活（7/1）した一方、再展開に合わせて入った新しいサイバーセキュリティ分類器が「無害な要求も従来より少し多めにフラグする」ため、コーディング中に一部が Opus へフォールバックする摩擦も生まれている。Claude Code 本体は 2.1.198 で Claude in Chrome を一般提供にし、バックグラウンドエージェントが作業完了後に自動で commit・push・ドラフト PR まで作るようになった。サブエージェントも次バージョンから既定でバックグラウンド実行になる。他陣営では Cognition が Devin Fusion（ハイブリッドなモデルルーティング）と Devin Security Swarm を投入、Notion は対話的 HTML ブロックを追加、OpenAI は GPT-5.6 Sol ファミリーを Codex に限定プレビューで載せた。モデルの世代交代と「エージェントが自分で最後まで運ぶ」流れが同時に進んだ 1 週間だ。

Claude Code

Claude Sonnet 5 が Claude Code の新デフォルトに——1M コンテキストで Opus 4.8 に迫る

6 月 30 日、Anthropic は Claude Sonnet 5 を発表した。「これまでで最もエージェント的な Sonnet」を掲げ、計画を立て、ブラウザやターミナルといったツールを使い、数か月前なら上位の高価なモデルが必要だった水準の自律動作をこなす、とされる。ポイントはコストと性能のバランスで、Anthropic は「Sonnet 4.6 に対して推論・ツール利用・コーディング・ナレッジワークすべてで大きく改善し、性能は Opus 4.8 に近い。それをより安い価格で」と説明する。国内エンジニアの整理でもベンチマーク上の序列は Opus 4.8 > Sonnet 5 > Sonnet 4.6 で、最上位に肉薄する位置づけだ。

プロダクト面で効くのは、これが Claude Code の既定モデルになった点だ。Free / Pro では既定、Max・Team・Enterprise でも利用でき、Claude Platform（API・Managed Agents）にも即日展開された。ネイティブで 1M トークンのコンテキストを持ち、長いコードベースを一気に読ませられる。8 月 31 日まで $2 / $10 per Mtok の導入価格が付く。早期アクセスのパートナーからは「これまでの Sonnet が途中で止まっていた複雑なタスクを最後までやり切る」「言われなくても自分の出力を検証する」といった反応が出ており、エージェント用途での“完走率”の改善が体感されている。リリースとしては Claude Code 2.1.197 が 1M コンテキスト対応込みで Sonnet 5 を既定化した。

Fable 5 復活——ただし新分類器が「無害なコーディング」も少し多めにフラグする

6 月 30 日、Anthropic は米商務省が Claude Fable 5・Mythos 5 への輸出規制を解除したと明かし、翌 7 月 1 日に Fable 5 が Claude Platform・Claude Code・Claude Cowork・Claude.ai へ復帰した。復帰に合わせて 5 時間・週次のレート制限もリセットされ、待たされていたユーザーがすぐ使い始められるようにした。「Fable is back」の一言が 7.6 万いいねを集めるなど、待望されていた復活だ。

ただし今回の再展開には 新しいサイバーセキュリティ分類器が組み込まれている。Anthropic は「より多くのサイバーセキュリティ関連タスクを狙って遮断する」ためのもので、「当面は無害な要求も従来の Fable より少し高い割合でフラグされる」と明言した。中の人（@trq212）も「ルーチンなコーディング・デバッグのごく一部がフラグされ、Opus にフォールバックする」と補足しており、コーディング中に自分のリクエストが弾かれる場面が起こりうる。Claude Code では /feedback、Claude.ai / Cowork では👍👎で誤検知を報告でき、そのフィードバックで分類器を調整し偽陽性を減らしていく、という運用だ。なお使用量込みの有料プランは 7 月 7 日まで、週次上限の 50% までを Fable 5 に充てられる（以降は他モデルに切替、またはクレジット消費）。

この“摩擦”を織り込んだ実践知も早速共有されている。@oikon48 は「Fable 5 に計画させ、実行は Opus 4.8 や Sonnet 5 に回す」構成を勧め、具体的には CLAUDE_CODE_SUBAGENT_MODEL でサブエージェントを Fable 以外に固定し、--append-system-prompt で Fable 運用向けの指示を足すやり方が今のところしっくりくる、としている。強力だが弾かれやすいモデルを“頭脳”に置き、実装は別モデルに任せる分業だ。

  • @claudeai: Fable 5 is back
  • @ClaudeDevs: 5 時間・週次のレート制限をリセット
  • @trq212: ルーチンなコーディングの一部はフラグされ Opus にフォールバック

2.1.193〜2.1.198——Chrome 一般提供、バックグラウンドエージェントが自動で PR まで作る

Claude Code 本体は今週も高頻度でリリースが続き、節目は 7 月 1 日の 2.1.198（32 の CLI 変更）だった。Claude in Chrome が一般提供となり、インストール不要でブラウザからセッションとエージェントに直接アクセスできる。さらに バックグラウンドエージェントが、コード作業を終えると worktree 上で自動的に commit・push し、ドラフト PR まで開くようになった。Notification フック（agent_needs_input / agent_completed）でバックグラウンドエージェントの完了・入力待ちを通知でき、チャート設計向けの /dataviz スキルも追加。組み込みの Explore エージェントはメインセッションのモデル（上限 opus）を継ぐようになり、サブエージェントも extended thinking の設定を継承する。

ここに至る流れは @bcherny の予告どおりだ。「次バージョンではサブエージェントが既定でバックグラウンド実行になり、動いている間もそのまま Claude と会話を続けられる。前面で走らせたければそう伝えればいい」——エージェントを“投げて待つ”のではなく“投げて会話し続ける”前提に寄せている。その他の直近リリースでは、2.1.196 が組織の既定モデル設定・クリック可能なファイル添付を入れ、セキュリティ面で「自己承認したリポジトリでは .mcp.json のサーバーを自動起動しない」よう塞いだ。2.1.193 は Bash/PowerShell を一律で auto-mode 分類器に通して安全確認を統一し、拒否理由をトランスクリプトと /permissions に記録して監査できるようにしている。リリースの一次情報は GitHub Releases（anthropics/claude-code）を参照したい。

  • @bcherny: 次バージョンでサブエージェントが既定でバックグラウンド実行に
  • @ClaudeCodeLog: 2.1.196——組織の既定モデル・.mcp.json の自動起動を制限
  • @ClaudeCodeLog: 2.1.193——Bash/PowerShell を auto-mode 分類器に一律通過

その他プロダクトの動向

Devin が Fusion と Security Swarm を投入——「ベンチではなくマージできるコード」を狙う

Cognition は 6 月 29 日、エージェント的コーディング向けのハイブリッド・モデルハーネス Devin Fusion を発表した。売り文句が挑発的で、「従来のモデルルーティングはダメだ。ベンチマークは通るが、実際にマージしたくなるコードは書けない」と切り出す。Fusion は複数モデルを組み合わせて、Fable 級の知性のコストを 35% 削減しつつ使い心地を落とさない、と主張する。モデルの単純な切替（ルーティング）が実務品質を担保しない、という問題意識は、前項の「Fable に計画させ実行は別モデル」という個人の工夫とも響き合う。

7 月 1 日には Devin Security Swarm も発表。Agentic MapReduce と名付けた新アーキテクチャで、複雑なコードベースからセキュリティ脆弱性をより低コストかつ高精度に見つける、というもの。大規模コードを分割して並列に探索し集約する MapReduce 的な発想を、エージェント群に持ち込んだ格好だ。コーディングエージェント各社が「書く」だけでなく「検証・防御」の側でも競い始めている。

  • @cognition: Devin Fusion——ハイブリッドハーネスで Fable 級のコストを 35% 削減
  • @cognition: Devin Security Swarm——Agentic MapReduce で脆弱性を探索

Notion に対話的 HTML ブロック——AI が説明コンテンツをその場でインタラクティブ化

7 月 1 日、Notion がページ内に HTML ブロックを追加した。Notion のページ上で対話的な HTML をそのまま構築でき、AI に頼めば手元のコンテンツを対話的な解説（explainer）・プロトタイプ・図に変換してくれる。チームで共有してその場で触りながら詰められる。先週の External Agents（各社のエージェントを同僚として招く）に続く動きで、Notion を「エージェントが生成した成果物を、チームで見て議論する場」に寄せる方向だ。実際、コーディングエージェントに埋め込み HTML ウィジェット付きの解説ドキュメントを書かせて共有する、といった使い方も共有されている。プロダクト視点では、エージェントの出力先が「コード／PR」だけでなく「チームで触れる対話的ドキュメント」へ広がっている点が面白い。

  • @NotionHQ: New block in Notion——対話的 HTML を AI で生成
  • @geoffreylitt: コーディングエージェントに埋め込み HTML 付き解説ドキュメントを書かせる

OpenAI が GPT-5.6 Sol ファミリーを Codex に限定投入、Codex の評価も上昇中

6 月 26 日、OpenAI は次世代フロンティアの GPT-5.6 Sol、日常向けにバランスした Terra、高ボリューム向けに高速・低価格な Luna の 3 モデルを発表した。ただし当面は米政府の要請で、Codex と API の一部信頼パートナー向けの限定プレビューにとどまる。Sam Altman は「Sol は GPT-5.5 と同価格、Terra は 5.5 水準を半額で」と価格面を強調し、7 月には Sol に 750 tok/秒が来るとも予告した。Sol はコマンドライン系のワークフローを測る Terminal-Bench 2.1 で SOTA を主張しており、Codex のようなターミナル常駐エージェントに効く方向の強化だ。

プロダクトとしての Codex 自体の評価も上向いている。Greg Brockman は外部の「Codex を侮っていた、実際かなり強い」という声を引いて「Codex はとても良くなった」と反応。Codex CLI も 0.142 系で細かく更新が続き、MCP のツール検索が既定でオン、macOS/Windows のシステムプロキシ（PAC / WPAD 自動検出）対応などが入った。基盤モデルの世代交代（Sol）と、それを載せる実行環境（Codex）の地味な作り込みが両輪で進んでいる。

  • @OpenAI: Sol が Terminal-Bench 2.1 で SOTA
  • @sama: Sol は 5.5 と同価格、Terra は 5.5 水準を半額で
  • @gdb: 「Codex はとても良くなった」
  • @Codex_Changelog: Codex CLI 0.142.2——MCP ツール検索が既定でオン、macOS プロキシ対応

コミュニティから

役割が溶け合う先の「原型」と、Spotify の 73% が AI 支援 PR という現実

エージェント前提で開発の役割そのものが変わりつつある、という議論が続いている。@bcherny は「エンジニアリング・プロダクト・デザイン・データサイエンスといった職能が新しい 1 つの役割に溶け合いつつある」とし、Claude Code チームを見ていると 5 つの原型（archetype）が見える、と投稿した。その筆頭が「Prototyper（新しいアイデアを次々に形にする人）」で、職種名ではなく“動き方”で人を捉え直す視点が 1.9 万いいねを集めた。ツールが仕事を吸収していく中で、人間側にどんな型が残る／生まれるのかを言語化する動きだ。

現場の数字も出てきている。Claude Code 開発者の Boris が Spotify のエンジニアリング担当 VP と対談し、Spotify は 1 日 4,500 回の本番デプロイ、PR の 73% がすでに AI 支援だと明かした。国内エンジニアの要約によれば、Niklas 氏はエージェントを成功させる鍵を「AI そのものよりエンジニアリングの基礎（テストの自動化など）」に置いており、派手なモデル論より足回りが効く、という現実的なメッセージが共有されている。エージェントを本番運用に乗せている組織のリアルな比率として、注目された。

  • @ClaudeDevs: Spotify は 1 日 4,500 デプロイ、PR の 73% が AI 支援
