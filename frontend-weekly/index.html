まとめ

今週はメタフレームワークとランタイムが「サーバーの速さで、クライアントの手触りを」を競った一週間。Vercel は 6 月 24 日に Next.js 16.3 のプレビューを公開し、Server Components を捨てずに SPA 並みの体感を狙う Instant Navigations を投入。翌 25 日には Deno 2.9 が deno desktop で「Web スタックのまま単一バイナリの native アプリ」を実験的に解禁した。先週 RC（6 月 18 日）が出た TypeScript 7.0 は GA（7 月中旬目安）の最終局面に入り、Vite も 8.1.0 で server.fs.deny を広げて足元を固めた。フレームワーク選びの軸は「どのリアクティビティか」から「ナビゲーションとビルドの体感をどう詰めるか」へ移りつつある。

今週の重要ポイント

Next.js 16.3 プレビュー — Instant Navigations で SPA の手触りを取り戻す

6 月 24 日、Vercel が Next.js 16.3 のプレビューを公開した。目玉は Instant Navigations。サーバーレンダリングの「ネットワーク待ち」で SPA に見劣りしていた遷移を、ルートごとに 3 つの挙動から選んで詰める設計だ。Suspense でローディングシェルを即時ストリーミングする Stream、'use cache' 済みの UI を即返す Cache、従来どおり待たせる Block をルート単位で指定できる。

もう一つの軸が Partial Prefetching。これまではリンクごとに prefetch していたため、サイドバーにチャットリンクが 20 本あれば 20 回リクエストが飛んでいた。新方式はルートごとに再利用可能なシェルを 1 度だけ取得してクライアントにキャッシュし、クリック時はシェルが即描画され残りがストリームで埋まる。開発支援として遅い遷移を開発時エラーで知らせる Instant Insights、DevTools の Navigation Inspector、Playwright 用の instant() ヘルパも揃う。インストールは next@preview から。

  • Next.js 16.3: Instant Navigations — nextjs.org
  • Next.js 16.3 preview adds instant navigations and partial prefetching — AlternativeTo

Deno 2.9 公開 — deno desktop で Web スタックから native アプリ

6 月 25 日に Deno 2.9 がリリースされた。看板機能は deno desktop。Electron のボイラープレート無しで、UI は webview、ロジックは Deno で動かし、全体を単一の配布バイナリにコンパイルする。出力は OS ネイティブ形式のインストーラ（macOS は .app/.dmg、Windows は .exe/.msi、Linux は .AppImage/.deb/.rpm）で、Next.js・Astro・Fresh・Remix・Nuxt・SvelteKit・SolidStart・TanStack Start・Vite SSR のフレームワーク自動検出に対応する。

移行まわりも進んだ。deno install が npm・pnpm・yarn・Bun のロックファイルを直接読めるようになり、パッケージマネージャの乗り換えが「移行作業」ではなく数コマンドで済む。hello-world のコールドスタートは 2.8 の 34ms から 17ms へほぼ半減した。なお deno desktop は 2.9 時点では experimental で、一部のプラットフォーム機能はなお実装途上である点に注意したい。

  • Deno 2.9 — deno.com
  • Deno project is going to add cross-platform desktop apps — The Register

TypeScript 7.0、GA 最終局面 — エディタ統合も前進

先週 RC（6 月 18 日）が出た Go ネイティブの TypeScript 7.0 は、RC から約 1 か月後とされる GA（7 月中旬目安）に向けて検証フェーズへ入った。型チェックは 6.0 比でおおむね 10 倍速とされ、「書き直し」ではなく「移植」なので型の意味論は従来と同一を保つ。RC は npm から即入手でき、1 年超のプレリリース検証を経て本番評価に十分耐えるという位置づけだ。

足元ではエディタ統合も動いた。Visual Studio 2026（18.6 Insiders 3）で TypeScript 7 ベータがデフォルト有効になり、IDE 側の既定が新コンパイラへ寄り始めた。本番採用前の検証を始めるなら、RC とエディタ更新が揃った今が好機。

  • TypeScript 7 Beta Now Enabled by Default in Visual Studio 2026 — Visual Studio Blog

フレームワーク・ランタイム

Turbopack の地ならし — メモリと永続キャッシュ、Rust 版 React Compiler

Next.js 16.3 は遷移だけでなくビルド基盤も底上げした。Turbopack には、長時間の開発セッションでコンパイラのメモリを回収する Memory Eviction、連続ビルドを速める Persistent Cache for Builds、Rust で再実装された React Compiler、import.meta.glob 対応が入った。日々の dev が重くなりにくく、CI のビルドも積み増しが効く方向だ。

AI/エージェント連携も今号の特徴。AGENTS.md 経由でバージョン整合したドキュメントをエージェントに読ませる仕組み、複数ステップを駆動する First-party Skills、React の内部を覗ける Agent Browser、対処方法を示す actionable errors、より小さく焦点を絞った MCP サーバなどが並ぶ。フレームワークが「人間と LLM の両方が読む対象」へ寄っていく流れが、メタフレームワーク側でも具体化してきた。

  • Next.js 16.3: Instant Navigations — nextjs.org
  • Next.js 16.3: Instant Navigations and Agent DevTools Are Here — byteiota

エコシステム・ツール

Vite 8.1.0 — Rolldown 追従と server.fs.deny の強化

6 月 23 日に Vite 8.1.0 が出た。同梱の Rolldown を 1.1.2 へ更新しつつ、開発サーバの server.fs.deny（配信を拒否するファイルのリスト）に一般的な機微ファイルを追加して既定を厳しくした。3 月に Vite 8 が Rolldown を単一の Rust 製バンドラとして本採用して以降、本体はパッチで安定度とセキュリティ既定を詰める局面に入っている。

VoidZero の Cloudflare 傘下入りという統治面の動きが続くなか、利用側として見るべきは「日々のリリースが止まらず、既定が安全側へ寄っているか」。今回の server.fs.deny 拡張はその一例で、アップデートはそのまま取り込んで問題になりにくい性質のものだ。

デザイン・CSS

Baseline 月次ダイジェスト — プラットフォーム標準への移行が続く

CSS/Web プラットフォーム側は、ライブラリで補ってきた領域がブラウザ標準へ移る流れが今月も続いている。直近の Baseline では image-rendering（拡大縮小時のスケーリングアルゴリズム制御。ピクセルアートや QR コードに有効）、下線がグリフを避ける挙動を強制する text-decoration-skip-ink: all、同一オリジンの複数コンテキストから使える SharedWorker API などが「Newly available」に加わった。

2026 年の Interop では anchor positioning（純 CSS で要素を別要素に係留＝Floating UI / Popper.js が担っていた仕事）や container style queries が中心テーマに据わる。採用判断は webstatus.dev などで各機能の対応状況を確認しながら進めたい。
