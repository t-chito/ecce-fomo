まとめ

今週は「ツールチェーンがネイティブへ渡り切った週」だった。先週 RC だった Go ネイティブの TypeScript 7.0 が 7 月 8 日に GA し、フルビルドで 8〜12 倍、VS Code の型検査は 125 秒から約 11 秒へ短縮された。標準の tsc がそのままネイティブ実装になり、型の意味論は 6.0 と同一のまま速さだけが載る。ただし Vue・Svelte・Astro・MDX のエディタ完全対応はプログラマティック API を待つ 7.1 送りで、単純アップグレードとはいかない点が今号最大の注意所だ。Vite 8 / Rolldown・oxc（Rust）に TypeScript（Go）が合流し、フロントのビルド基盤は「JS で書かれたツールを別言語のネイティブ実装へ置き換える」波が主軸になった。同じ 7 月 8 日には Node.js 26.5.0 も出て、Temporal 既定化・年 1 メジャーへの移行と足並みが揃う。競争軸は「どのフレームワークか」から「型検査・ビルド・起動のフリクションをネイティブでどこまで削るか」へ移っている。

今週の重要ポイント

TypeScript 7.0 が GA — Go ネイティブの tsc、フルビルド 8〜12 倍

先週 RC だった Go ネイティブ版 TypeScript が、7 月 8 日に 7.0 として GA した。Go への移植（書き直しではなく port）により型検査ロジックは 6.0 と構造的に同一で、コンパイラが強制する意味論は従来と変わらない。にもかかわらず、ネイティブコード速度と共有メモリのマルチスレッド化でフルビルドはおおむね 8〜12 倍速く、VS Code コードベースの型検査は 125 秒から 11 秒未満（約 11.9 倍）へ落ちた。配布は実験的だった tsgo という別名ではなく、標準の tsc バイナリがそのままネイティブ実装として出る形に収束した。

ただし本番投入前に一点押さえたい。Vue・Svelte・Astro・MDX のエディタ完全対応は 7.0 には間に合わず、プログラマティック API の実装を待つ 7.1 送りになった。これらのフレームワークは型情報を得るために TypeScript の API を叩くため、単一ファイルコンポーネントの補完・型チェックがエディタで十全に効くのは次のマイナー以降になる。素の .ts/.tsx 主体のプロジェクトは今すぐ恩恵を受けられるが、SFC 中心のスタックは 7.1 を待つか、当面は 6.x 系のエディタ体験を併用する判断になる。1 年超のプレリリースを経た本命リリースだけに、この配布の谷を正しく見切ることが移行の要だ。

ネイティブ・ツールチェーンの合流 — Go の型検査に Rust のビルド系が揃う

TypeScript 7.0 の GA は単発の速さ自慢ではなく、ここ 1 年進んできた「JS で書かれたフロントツールを別言語のネイティブ実装へ置き換える」流れの節目にあたる。バンドラは Vite 8 が Rolldown（Rust）を単一の既定バンドラとして取り込み済みで、Rolldown の変換パイプラインや Vite 8 の React プラグインには oxc（Rust 製の JS/TS パーサ・変換・lint 基盤、VoidZero が Vite/Rolldown と一体で開発）が組み込まれている。lint も oxlint・Biome という Rust 製が ESLint の置き換え候補として実運用段階に入った。

今週これに TypeScript（Go）の型検査が正式合流したことで、パース・変換・バンドル・lint・型検査という主要工程がひと通りネイティブ実装で揃った。言語こそ Go と Rust に分かれるが、狙いは共通で「大規模コードベースで CPU とメモリと待ち時間を削る」ことだ。個々のツールの速度比較より、フロントの標準ツールチェーン全体が数年がかりでネイティブへ移り終えつつある、という構図が週単位の読みどころになる。

Node.js 26.5.0 — 同じ 7 月 8 日、Temporal 既定化と年 1 メジャーへ

Node.js も 7 月 8 日に 26.5.0（Current）を出した。パッチ号としては blob.textStream() の追加、テキストを直接インポートする --experimental-import-text、イベントループ 1 反復ごとの遅延サンプリングなど小粒な改善が中心だが、背景の Node.js 26 系そのものが節目だ。Temporal API が既定で有効になり、V8 は 14.6、Undici は 8.0 に上がり、旧来の非推奨 API が整理された。

リリース運用も転換点にある。Node.js 26 は 10 月に LTS へ入るまで約半年 Current を務めるが、これが従来の奇数/偶数スケジュール最後の号で、Node 27 以降は年 1 回のメジャーへ移る。ネイティブ・ツールチェーンが速さを競う一方で、ランタイム側は標準 API の拡充（Temporal）と予測可能なリリース間隔という「地固め」に軸足を置いている。

ビルドツール・ランタイム

Next.js 16.3 プレビュー — Turbopack がメモリ最大 90% 減・永続キャッシュ・Rust 版 React Compiler

6 月 29 日に出た Next.js 16.3 のプレビュー（next@preview で導入、安定版はまだ）は、native toolchain の波が Next 本体にも及んでいることを示す。目玉は Turbopack で、大規模アプリの開発サーバのメモリ使用を最大 90% 削減し、永続ビルドキャッシュで本番ビルドを最大 5.5 倍高速化した。あわせて React Compiler の実験的な Rust 移植版と、Vite 互換の import.meta.glob が入った。

長い開発セッションでコンパイラのメモリを回収するメモリエビクション、ビルド間で再利用できる永続キャッシュと、狙いは TypeScript 7.0 や Rolldown と同じ「大規模での待ち時間とリソースを削る」ことに一致する。React Compiler まで Rust へ移植し始めた点は、フロントのビルド系がフレームワークの内側までネイティブ化している証左だ。まだプレビューなので、評価は next@preview で行い、本番は安定版を待つのが筋になる。

ブラウザ・プラットフォーム

Interop 2026 の中間状況 — ブロック要因は Safari 59・Firefox 43・Chromium 9

Chrome・Firefox・Safari・Edge の 4 陣営が協調する Interop 2026 は、コンテナ要素、ハイライト文字のカスタムスタイル、ローカルデータベース、WebAssembly、スクロールスナップ、View Transitions、WebRTC などを対象に、年末までに主要ブラウザで揃えることを目標に進む。前年の Interop 2025 はスコアが 25 から 95 まで伸び、Same-Document View Transitions・CSS Anchor Positioning・Navigation API・CSS @scope・URLPattern といった大物がクロスブラウザで使えるようになった実績を残した。

7 月時点の分析では、機能が「広く利用可能（Widely available）」になるのを妨げているブロック要因が、6 月 30 日時点で Safari が単独で 59 件、Firefox が 43 件、Chromium が 9 件と偏る。CSS テストの通過率は Firefox 84.9%、Safari 86% と拮抗しつつも、どのエンジンが何を止めているかで採用可能な機能の地図が変わる。前掲の DevTools への Baseline 表示（Chrome 140）と合わせ、「いつ・どのブラウザで安全に使えるか」を実装の脇で判断できる材料が今年は着実に増えている。
