まとめ

今週は「ツールチェーンの合流」が通底したテーマ。Svelte の 7 月号は SvelteKit の設定を Vite プラグインへ直接渡せるようにし、svelte.config.js という別ファイルを不要化する方向を打ち出した。設定・言語ツール・CLI が SvelteKit 3 の姿へ揃っていく動きだ。先週 RC（6 月 18 日）が出た Go ネイティブの TypeScript 7.0 は GA 秒読みに入り、7 月 5 日にも改めて技術メディアが取り上げた。開発ツール側では Chrome 140 の DevTools が Styles ペインに Baseline 表示を載せ、「このプロパティは使ってよいか」をエディタから引かずに判断できるようになった。フレームワークの競争軸が「どのリアクティビティか」から「設定・型検査・採用判断のフリクションをどこまで削るか」へ移りつつある。

今週の重要ポイント

Svelte 7 月号 — SvelteKit の設定を Vite プラグインへ、svelte.config.js を不要化

7 月の「What's new in Svelte」で、SvelteKit の設定をそのまま Vite プラグインへ渡せるようになった。これまで独立して置いていた svelte.config.js が必須ではなくなり、Vite の設定に一本化できる。設定ファイルが二重化していた状態が解消され、ビルドツールの中心である Vite の側へ寄る流れが明確になった。

あわせて明示的な環境変数（explicit environment variables）が導入され、これは SvelteKit 3 で $env/* モジュールがどう振る舞うかのプレビューにあたる。言語ツールと sv CLI も Svelte の新しい {@const ...} 宣言タグに対応し、コンパイラ・エディタ・CLI がひと揃いで足並みを揃えた。個々は小粒でも、SvelteKit 3 に向けて土台を整地する一連の動きとして読める。

  • What's new in Svelte: July 2026 — svelte.dev

TypeScript 7.0、GA 秒読み — RC から GA へ、tsgo から tsc へ

6 月 18 日に RC が出た Go ネイティブの TypeScript 7.0 は、RC から約 1 か月後とされる GA（7 月中旬目安）に向けた検証フェーズが続く。7 月 5 日には Slashdot をはじめ技術メディアが改めて「ついに RC に到達」と取り上げ、1 年超のプレリリースを経た本番評価フェーズであることを再確認した。型チェックは 6.0 比でおおむね 10 倍速とされ、パース・型チェック・出力を並列に走らせる点が速さの源だ。

移行者が押さえるべきは配布経路の整理。RC は標準の tsc バイナリで配られ、実験的に配られてきた tsgo という別名は nightly 側に残る形になった。「書き直し」ではなく「移植」なので型の意味論は従来と同一を保つ。本番採用前の検証を始めるなら、RC が npm から即入手できる今が好機だ。

  • Go-based TypeScript 7.0 Finally Reaches Release Candidate Stage — Slashdot
  • Go-based TypeScript 7.0 reaches release candidate stage — InfoWorld

Chrome 140 DevTools — Styles ペインに Baseline 表示、採用判断をその場で

Chrome 140 の DevTools で、Elements パネルの Styles セクションに Baseline 情報が載った。CSS プロパティにホバーすると、そのプロパティが各主要ブラウザでどの Baseline 段階（Newly available / Widely available）にあるかがカードで表示される。Widely・Newly のカードには相互運用が始まった日付も示され、「いつから安全か」をその場で確認できる。

これまで webstatus.dev や MDN を別タブで引いていた「このプロパティは本番で使ってよいか」の判断が、書いているスタイルの脇で完結する。CSS の新機能がプラットフォーム標準へ移る流れが続くなか、採用判断のフリクションを削るツール側の後押しとして意味が大きい。Chrome 140 はアナウンス時点でベータ、まもなく安定版へ降りる段階だった。

  • Baseline for CSS properties now in Chrome DevTools — web.dev

フレームワーク・ランタイム

Deno 2.9.1 — deno desktop の追い込みパッチ

先週（6 月 25 日）に deno desktop で「Web スタックのまま単一バイナリの native アプリ」を実験的に解禁した Deno は、7 月 1 日に 2.9.1 のパッチを出した。desktop 関連の追加・修正が中心で、experimental として出した機能を実利用に耐える形へ詰める段階に入っている。hello-world のコールドスタートが 2.8 の 34ms から 17ms へほぼ半減した性能面の改善はそのまま引き継ぐ。

deno desktop は 2.9 時点で experimental の位置づけであり、一部プラットフォーム機能はなお実装途上。今回のような小刻みなパッチで安定度を上げていく局面なので、試すなら最新パッチに追従しておきたい。

  • Release v2.9.1 · denoland/deno — github.com
  • Deno 2.9 — deno.com

Angular の「signal-first」が既定に — v22 の定着

6 月 3 日に出た Angular 22 は、Angular 21 で experimental だった Signal Forms を stable に格上げし、新規プロジェクトの既定を signal 中心（zoneless・OnPush 既定、テストランナーは Vitest）へ振り切った「signal-first」リリースだった。今週はその路線が実プロジェクトへ浸透していく地固めの週で、React・Svelte が進めてきた signal / runes ベースのリアクティビティに Angular も既定として合流した格好だ。

Reactive Forms からの移行や zone.js 撤去の schematics が揃っているため、既存アプリの評価を始めるハードルは下がっている。フレームワーク横断で「明示的なリアクティビティ」が共通言語になりつつある点が、週単位の動向として読み取れる。

  • What's new in Angular 22.0? — Ninja Squad
  • Angular 21 – What's New? — angular.love

ライブラリ・UI

Ext JS 8.0 GA — 電子署名・QR・Font Awesome 7・ES2025

Sencha が Ext JS 8.0 を GA としてリリースした。140 超のプリインテグレート済み UI コンポーネント群に、アプリ内で書類に署名できる Responsive Digital Signature Pad と、決済・小売・物流の QR ワークフロー向けの QR Code Reader & Generator（いずれも Classic / Modern 両トールキット対応）を追加した。

アイコンは Font Awesome 7.x を既定に採用（FA5 と後方互換）、言語サポートは ES2025 まで拡張し、グリッドの水平バッファリングや Lockable Grid Plugin、Modern トールキットの ARIA 対応も入った。エンタープライズ用途のフルスタックな UI キットとして、書き直し不要の後方互換を保ったままの底上げが今号の性格だ。
