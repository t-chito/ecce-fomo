# 週間プロダクトリリースノート

紙名: `プロダクトリリースノート`
補助ラベル: `過去1週間`

## 調査依頼

過去 1 週間における監視対象プロダクトのリリースノートを調べ、週単位で読みやすい形に整理してまとめる。

## 監視対象

| プロダクト | URL |
| --- | --- |
| Claude Code | `https://github.com/anthropics/claude-code/releases` |
| Codex CLI | `https://github.com/openai/codex/releases` |
| Gemini CLI | `https://github.com/google-gemini/gemini-cli/releases` |

## 情報取得のルール

- 元のリリースノート（GitHub Releases の各バージョンページ）への URL を必ず含める
- 各リリースのバージョン番号と公開日時を記載する
- 監視対象に該当しないプロダクトのリリースは扱わない

## 出力構造

冒頭にまとめを置き、続いてプロダクトごとのセクションを並べる。

- まとめ: 今週の最大の注目点（プロダクト横断で）を 3〜5 行で。冒頭に配置し、忙しいときはここだけ読めば全体を把握できるようにする
- プロダクトごとのセクション: 監視対象の各プロダクトに 1 セクション
  - 冒頭に「特に重要なリリース内容」のサマリを散文で置く（その週のそのプロダクトの目玉を整理）
  - その下に、各リリースのバージョン・公開日・変更内容を時系列順（新しい順）に列挙する。各リリース見出しから対応するリリースノートにリンクする
- 新着リリースが 0 件のプロダクトはセクション内に「今週は新着なし」とだけ記す

## 週次ならではの編集方針

- 同じ機能・修正に関する複数の進捗（プレビュー → 正式、修正の再修正など）は、リリースごとに分けつつも「特に重要なリリース内容」サマリで流れが追えるようにする
- 軽微な内部変更・依存更新は、各リリースの変更内容では網羅しつつ、サマリでは触れない
- 重大な不具合修正・破壊的変更は、まとめおよび各プロダクトのサマリで必ず触れる
