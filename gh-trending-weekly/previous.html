まとめ

今週の週間トレンドは、先週まで主役だった「エージェントを動かすハーネス」から、エージェントに渡す中身——スキル集・テンプレート・仕様フォーマット——へと重心が広がった。Claude Code / Codex を価値投資の研究フレームワークに仕立てた xbtlin/ai-berkshire が週間 +7,000★ 近くで急上昇し、232 のエージェント人格を束ねた agency-agents、コーディングエージェントに Web サイトを複製させる ai-website-cloner-template、デザインシステムをエージェント可読にする Google Labs の design.md が続いた。先週首位の calesthio/OpenMontage は +1 万★ で依然トップ級を維持。エージェントの「記憶」層でも DeusData/codebase-memory-mcp が今週も +9,800★ と伸び、topoteretes/cognee が並んだ。AI 以外では、識別子を一切持たないメッセージング simplex-chat と、無料枠サービスの定番リスト free-for-dev、自前ホストの旅行計画 TREK が安定した支持を集めた。

エージェントに渡す「中身」——スキル・テンプレート・仕様

xbtlin/ai-berkshire — Claude Code / Codex で組む価値投資リサーチ框架（週間 +6,989★）

今週最大の伸びを見せた新顔。Warren Buffett・Charlie Munger らの価値投資の方法論を、Claude Code / Codex 上で回す研究フレームワークとして実装したもの。企業の深掘り調査・決算レビュー・業界スクリーニング・ポートフォリオ管理・投資チェックリストなど、18 の専門「スキル」を備える。

特徴は、単一視点の AI 分析に頼らず、複数のエージェントを異なる投資哲学のもとで並列に走らせ、各投資案件を四つの競合する視点から批判的に検討する「マルチエージェント敵対分析」にある。「合格／不合格／グレーゾーン」を明示する構造化された意思決定フレームと、計算のクロスバリデーションを行う財務ツールを組み合わせる。主体は Python、ライセンスは MIT。エージェントの器ではなく、特定領域の「思考の型」をスキルとして与える方向を象徴する動きだ。

エンジニアリング・デザイン・マーケティング・営業・プロダクト・セキュリティ・財務・ゲーム開発・GIS など 16 部門にまたがる、232 の専門 AI エージェント人格を収めたコレクション。README は「指先に完結する AI 代理店」と謳う。汎用プロンプトではなく、各エージェントに固有のプロセス・成果物・成功指標を持たせている点を特徴とする。

Claude Code・GitHub Copilot・Cursor・OpenCode など 10 以上のツールへネイティブに組み込め、ワンクリック導入のデスクトップアプリ（macOS／Linux／Windows）とマルチツール変換スクリプトを備える。主体は Shell（84.7%）と Python、ライセンスは MIT。前週の各種「スキル集」に続き、今度は職能別の「人格集」がまとめて流通し始めている。

JCodesMore/ai-website-cloner-template — サイトを Next.js に複製させるテンプレート（週間 +4,843★）

コーディングエージェントを使って、既存 Web サイトをクリーンな Next.js コードベースへ「リバースエンジニアリング」するための再利用テンプレート。URL を指定して /clone-website を実行すると、エージェントがサイトを解析し、デザイントークンとアセットを抽出、コンポーネント仕様を書き起こし、各セクションを並列に再構築する。

偵察→基盤構築→コンポーネント仕様→並列ビルド→組み立て、という多段パイプラインで動き、Claude Code・GitHub Copilot・Cursor・Windsurf など 11 以上のエージェントに対応する。技術スタックは Next.js 16・React 19・shadcn/ui・Tailwind CSS v4・TypeScript strict。プラットフォーム移行、失われたソースコードの復元、実運用サイトからの学習を想定用途に挙げる。ライセンスは MIT。

google-labs-code/design.md — デザインシステムをエージェント可読にする仕様フォーマット（週間 +6,240★）

Google Labs による、コーディングエージェントがデザインシステムを一貫して理解・適用できるようにするためのフォーマット仕様。機械可読なデザイントークン（YAML）と、人間可読な設計意図の記述（Markdown 散文）を一つのファイルに束ね、エージェントに「デザインシステムの永続的で構造化された理解」を与えることを狙う。

DESIGN.md ファイルの lint・diff・エクスポートを行う CLI ツール、WCAG コントラスト比チェックを含むトークン検証、Tailwind や W3C Design Token 形式へのエクスポート、構造・意味の誤りを捕らえる 9 つの lint ルールを備える。主体は TypeScript（96.1%）、ライセンスは Apache-2.0。現時点では alpha 段階。エージェントに「見た目のルール」を規格として渡す試みで、design.md という命名自体が README／AGENTS.md といった既存慣習の延長線上にある。

「いかなる種類のユーザー識別子も持たずに動作する、初のメッセージングネットワーク——設計から 100% プライベート」。ユーザー ID を割り当てず、二重ラチェット暗号（ポスト量子耐性を含む）と使い捨てのリレーノードによって、プラットフォーム側からも外部の観測者からも、利用者の身元・連絡先・メタデータを守ることを掲げる。

iOS／Android のモバイルアプリ、デスクトップ／ターミナルの CLI クライアントを備え、グループ・ファイル・音声／ビデオ通話、IP アドレスを隠すプライベートなメッセージ経路、任意の Tor 連携に対応する。主体は Haskell・Kotlin・Swift、ライセンスは AGPL-3.0。AI 一辺倒のトレンドの中で、プライバシー志向の通信基盤が根強い支持を示した。
