まとめ

今週の週間トレンドは、先週まで続いた「エージェントに渡す中身（スキル・仕様）」の潮流が一段進み、ベンダーの垣根を越えた連携とローカル志向の二軸に集約された。象徴は OpenAI 公式の codex-plugin-cc——Claude Code の中から Codex を呼んでレビュー・タスク委譲する、競合同士をつなぐプラグインが週間 +4,800★ 近くで急伸した。Meta はエージェント可読を掲げるデザインシステム astryx を公開し、Microsoft は dotnet/skills、Alibaba は page-agent と、大手が相次いでエージェント向け資産を投下している。基盤側では複数エージェントを束ねる herdr、隔離実行する CubeSandbox、231+ プロバイダを束ねる OmniRoute が伸びた。もう一方の軸はプライバシー・ローカルで、会議を端末内で完結させる meetily が週間 +8,885★ で今週の伸び頭、HuggingFace の speech-to-speech も続いた。先週上位の strix・orca・ai-berkshire は今週も残り、AI 一色の傾向は変わらない。

ベンダーの垣根を越えるエージェント連携

openai/codex-plugin-cc — Claude Code から Codex を呼ぶ OpenAI 公式プラグイン（週間 +4,792★ / 累計 2.7万★）

OpenAI が公式に公開した、Claude Code のワークフローの中から Codex を使うためのプラグイン。競合するコーディングエージェント同士をつなぐという構図が注目を集め、今週の話題をさらった。既存の作業環境を離れずに、コードレビューやタスクの委譲を Codex に任せられる。

/codex:review の通常レビュー、前提や設計判断に切り込む /codex:adversarial-review、デバッグや実装を丸ごと預ける /codex:rescue、Claude Code のセッションを Codex 側の永続スレッドへ移す /codex:transfer を備え、/codex:status・/codex:result・/codex:cancel でバックグラウンドジョブを管理する。Codex の検証が通るまで Claude の応答をブロックする任意のレビューゲートも持つ。ChatGPT サブスクリプション（無料枠を含む）または OpenAI API キーと Node.js 18.18 以上が必要。主体は JavaScript、ライセンスは Apache-2.0。「どちらか一方を選ぶ」ではなく「両方を協調させる」方向を、当のベンダーが後押しし始めた点が象徴的だ。

  • openai/codex-plugin-cc

facebook/astryx — 人とエージェントが同じ道具で作る Meta のデザインシステム（週間 +4,087★ / 累計 7.6千★）

Meta が公開したオープンソースのデザインシステム。「人とエージェントのために作られた」を掲げ、API・ドキュメント・CLI を一体で設計し、人間の開発者と AI アシスタントが同じ参照・同じ流儀で UI を組めるようにする。社内で最も使われる最大のデザインシステムとして育ったものを外部公開したという。

TypeScript 対応の 150 以上のアクセシブルな React コンポーネント、neutral・butter・chocolate・matcha など 7 つのテーマ、ダークモード、CSS カスタムプロパティによるブランド単位のテーミングを備える。コンポーネントのスキャフォールドやドキュメント生成を担う CLI を持ち、その CLI を支える API を AI エージェントやビルドツールも同じように叩ける。命名・prop・合成の規則を全コンポーネントで統一し、数個覚えれば残りの挙動を人も AI も予測できる設計を強調する。Tailwind・CSS modules・素の CSS のいずれとも併用可能で、リポジトリには CLAUDE.md／AGENTS.md を同梱する。現時点では beta、ライセンスは MIT。

  • facebook/astryx

dotnet/skills — .NET / C# 開発をエージェントに教える公式スキル集（週間 +886★ / 累計 4.5千★）

Microsoft の .NET チームによる、AI コーディングエージェントに .NET・C# プロジェクトを扱わせるためのスキルとカスタムエージェントのコレクション。C# の言語サーバー（LSP）連携を組み込み、データアクセス・診断・ビルド・パッケージ管理・アップグレード・MAUI・AI/ML・ASP.NET Core・Blazor・テストなど多領域の開発タスクをカバーする。

テストフレームワークの移行ツールやテンプレートエンジンによるプロジェクト雛形生成を備え、.NET 11 の機能にも対応する。agentskills.io のオープン標準に沿い、Copilot CLI・Claude Code・VS Code・Cursor・Codex CLI などのプラグインマーケットプレイス経由で導入できる。主体は C#、ライセンスは MIT。先週の各種「スキル集」に続き、大手プラットフォーマー自身が自社言語向けのスキルを公式に配り始めた例だ。

  • dotnet/skills

alibaba/page-agent — ページ内で動く JavaScript 製 GUI エージェント（週間 +4,459★ / 累計 2.6万★）

Alibaba による、Web インターフェースを自然言語で操作するためのページ内 GUI エージェント。ブラウザ拡張・Python・ヘッドレスブラウザを必要とせず、JavaScript を組み込むだけで動く点を特徴とする。スクリーンショットに頼らず、テキストベースで DOM を解析・操作する。

ローカル配置を含む複数の LLM に対応し、複数ページにまたがるタスク向けの任意の Chrome 拡張、beta の MCP サーバー対応を持つ。SaaS への AI コパイロット組み込み、ERP／CRM のスマートなフォーム入力、自然言語による Web アクセシビリティ、MCP クライアントからのブラウザ制御を想定用途に挙げる。主体は TypeScript、ライセンスは MIT。エージェントを Web ページの内側に住まわせる、ブラウザ操作系の新しい形だ。

  • alibaba/page-agent

エージェントを束ねる・隔離する基盤

ogulcancelik/herdr — 端末で複数エージェントを多重化するマルチプレクサ（週間 +4,756★ / 累計 1.5万★）

ターミナル環境向けのエージェント・マルチプレクサ。複数の AI エージェントを一つのターミナル画面の中で同時に管理し、セッションを切り離して再接続できる。ブロック中・作業中・完了といったエージェントの状態をリアルタイムに可視化する。

ターミナルを切り替えても状態を保つセッション永続化、エージェント自身がペインを生成して互いに協調できるネイティブなソケット API、キーボード・マウス両対応、拡張可能なプラグイン機構を備える。Electron に依存しない単一の軽量な Rust バイナリである点を売りにする。ライセンスは AGPLv3（組織向けの商用ライセンスも用意）。先週の orca に代表される「並列エージェントの艦隊を操る」路線が、より軽量なターミナル常駐ツールとして広がっている。

  • ogulcancelik/herdr

TencentCloud/CubeSandbox — AI エージェント向けの高速・隔離サンドボックス（週間 +2,284★ / 累計 9.5千★）

Tencent Cloud による、AI エージェントのための「即時・並行・安全・軽量」なサンドボックスサービス。LLM が生成した信頼できないコードを、隔離された仮想マシン内で安全に実行することを狙う。KVM ベースで専用のゲスト OS カーネルを用いるハードウェアレベルの隔離を提供し、60ms 未満のコールドスタートとインスタンスあたり 5MB 未満のメモリ増加を謳う。

E2B SDK 互換による移行のしやすさ、管理・監視の Web コンソール、API キーをサンドボックス外に保つ資格情報保管庫、ドメイン許可リストと監査ログによる egress 制御、スナップショット・クローン・ロールバック、コスト最適化のための自動一時停止／再開を備える。ARM64・x86_64 の双方に対応。主体は Rust・Go・C、ライセンスは Apache 2.0。エージェントに「手足」を与えるうえで避けて通れない実行環境の安全性に、クラウド事業者が本体を投入した格好だ。

  • TencentCloud/CubeSandbox

diegosouzapw/OmniRoute — 231+ プロバイダを束ねる無料 AI ゲートウェイ（週間 +4,119★ / 累計 1.5万★）

単一のエンドポイントから多数の AI プロバイダへリクエストを振り分ける、オープンソースの無料 AI ゲートウェイ。README は 237 プロバイダ（うち 90 以上が無料）に対応すると謳い、Claude Code・Cursor・Cline・Copilot などのコーディングツールを複数モデルへ接続し、自動フォールバックとトークン圧縮を効かせる。

優先度・重み付け・コスト最適化・フュージョンなど 17 のルーティング戦略、トークンを 15〜95% 削減するというスタック型の圧縮、95 ツールを持つ MCP サーバーと A2A エージェントプロトコル、チーム間でのクォータ共有と公平スケジューリング、PII 検出やプロンプトインジェクション対策のガードレールを備える。npm・Docker・デスクトップ（Electron）・Android（Termux）・PWA と配布形態も幅広い。主体は TypeScript、ライセンスは MIT。多プロバイダ時代のコスト最適化とレジリエンスを一枚で担う層として支持を集めた。

Zackriya-Solutions/meetily — 端末内で完結するプライバシー重視の会議アシスタント（週間 +8,885★ / 累計 2.2万★）

今週の伸び頭。会議の記録・文字起こし・要約を、クラウドに送らずローカルマシン上で完結させる、プライバシー重視の AI 会議アシスタント。Whisper や Parakeet を使ったリアルタイムのローカル文字起こしと、Ollama・Claude・Groq・OpenRouter に対応した AI 要約を組み合わせる。

macOS・Windows・Linux のマルチプラットフォームに対応し、GPU アクセラレーションを効かせた音声ミキシング、既存音声ファイルの取り込みと強化、OpenAI 互換エンドポイントのカスタム指定を備える。防衛・法務・医療・コンプライアンス領域など、機密性の高い会話の完全な統制を必要とする組織を主要な想定利用者に挙げる。主体は Rust・TypeScript・C++、ライセンスは MIT。クラウド前提の議事録 SaaS に対し、データ主権を軸にしたローカル完結型が強い支持を得た。

huggingface/speech-to-speech — オープンモデルで組む低遅延の音声エージェント（週間 +788★ / 累計 5.9千★）

Hugging Face による、オープンソースモデルだけで音声エージェントを組むためのパイプライン。「VAD → STT → LLM → TTS」というモジュール構成を取り、OpenAI Realtime 互換の WebSocket API を備える。各段を差し替え可能にし、完全にローカルで低遅延の音声対話を実現することを狙う。

STT には Parakeet TDT や Whisper 系、TTS には Qwen3-TTS や Kokoro、LLM にはローカルの transformers・mlx-lm・OpenAI 互換 API と、複数のバックエンドを選べる。リアルタイムサーバー・ローカル・WebSocket・TCP ソケットの複数モードで動き、多言語対応と言語自動判定、Pocket TTS による任意の音声クローンを持つ。Reachy Mini ロボット上で実運用されているという。主体は Python、ライセンスは Apache 2.0。テキスト系に続き、音声もローカル・オープンで組む流れが定着しつつある。

  • huggingface/speech-to-speech

bradautomates/claude-video — Claude に動画を解析させるフレーム抽出ツール（週間 +3,630★ / 累計 7.0千★）

動画をダウンロードしてフレームを抽出し、文字起こしと合わせて Claude に渡すことで、動画コンテンツを解析させるツール。/watch コマンドに動画の URL やローカルファイルを与えると、利用可能なら字幕（無料）を取得し、シーンを意識したフレームを設定可能な粒度で抜き出す。字幕が無い場合は Whisper API による音声書き起こしにフォールバックする。

動画の構成や「フック」の分析、スクリーン録画からのバグ診断、内容の要約、プレイリストの検索可能なノート化、全編を見ずに要点だけ抜き出す、といった用途を挙げる。主体は Python、ライセンスは MIT。テキスト中心だったコーディングエージェントに、動画という入力を安価に足す小道具として注目された。

  • bradautomates/claude-video

asgeirtj/system_prompts_leaks — 各社 AI のシステムプロンプトを集めた資料庫（週間 +7,149★ / 累計 5.6万★）

Claude・ChatGPT・Gemini・Grok など各社チャットボットのシステムプロンプトを抽出・収集して整理したリポジトリ。Anthropic・OpenAI・Google・xAI・Microsoft といった提供元ごとに分類し、モデルのバージョン間でプロンプトの変化を diff で追える。ツール仕様や連携用プロンプトも含む。

AI のシステム指示を理解するための教育的な資料として位置づけられ、累計 5.6万★ に達する定番アーカイブになっている。ライセンスは CC0-1.0（パブリックドメイン提供）。エージェント自体を作る動きが盛り上がる裏で、「既存のエージェントが実際どう指示されているか」を覗く AI 自己言及的なメタ資料への関心も根強い。

harvard-edge/cs249r_book — 「ML システム」を学ぶ Harvard の教材（週間 +1,969★ / 累計 2.7万★）

孤立したモデルではなく、本番で動く効率的・堅牢な知的システムを作る「AI エンジニアリング」を教える、統合型のカリキュラム。機械学習とシステム工学をつなぐ理論・メンタルモデルを扱う 2 巻構成の教科書を核に、学習者が ML の内部を 20 モジュールで自作する TinyTorch、設計トレードオフを探る Marimo ノートブックのインタラクティブ実習を備える。
