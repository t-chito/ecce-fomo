まとめ

先週は「AI にデザインの目を持たせる」流れが主役だったが、今週の週間トレンドはコーディングエージェント本体の新顔ラッシュに主役が移った。Moonshot が TypeScript 製の次世代 CLI kimi-code を投入し（既存の Python 版 kimi-cli はそこへ吸収されていく）、Rust で性能を極める harness jcode、そして LLM API からエージェントループ・TUI・CLI までを束ねる pi（＋その Web UI pi-web）が揃ってランクイン。エージェントを「どう名乗らせ、どう速く動かすか」の競争が一段と激しくなった。もう一つの軸は文脈の知能化で、今週の実質トップ code-review-graph（週間 +6,257★）はコードベースをグラフ化し、エージェントに「読むべき所」だけを渡して 38〜528 倍のトークン削減を謳う。デザイン系スキルの潮流も途切れず、先週の hallmark に ui-skills が加わった。そしてエージェントの外側でも、地政学ダッシュボード worldmonitor（今週最大の +9,054★）、3D シーン復元の基盤モデル lingbot-map、異種混合 LLM 推論の ktransformers が大きく伸び、AI 一色ながらも裾野の広がりを見せた週になった。

コーディングエージェントの新顔ラッシュ

MoonshotAI/kimi-code — Moonshot が投入する次世代エージェント CLI（週間 +1,610★ / 累計 4.9千★）

端末内で動く AI コーディングエージェント。コードの読み書き、シェルコマンドの実行、ファイル検索、Web ページの取得を行い、返ってきたフィードバックを見て次の一手を自分で選ぶ、という自律ループを核に据える。Moonshot（Kimi）の CLI エージェントを「次世代エージェントの出発点」と位置づけて打ち出したものだ。

単一バイナリでの最小依存インストール、画面録画やデモ動画をエージェントに読ませる動画入力、対話的に設定できる MCP サポート、並列で個別タスクを進めるサブエージェント、Agent Client Protocol（ACP）経由での Zed・JetBrains 連携、要所でコマンドを差し込むライフサイクルフック、拡張用のプラグイン機構を備える。主体は TypeScript、ライセンスは MIT。既存の kimi-cli がここへ収斂していく設計で、モデルベンダー自身が公式 CLI を磨く動きの新たな一例となった。

  • MoonshotAI/kimi-code

MoonshotAI/kimi-cli — kimi-code へ発展途上の Python 版 CLI（週間 +1,528★ / 累計 1.1万★）

同じ Moonshot による、端末ベースの AI エージェント。コードの読み書き・シェル実行・Web 取得を行い、目的達成に向けて実行中に方針を自律調整する。README 自身が「Kimi CLI は Kimi Code CLI へと進化している」と明言しており、今週上位に並んだ kimi-code の前身・並走版という立ち位置にある。

Ctrl-X でアプリを抜けずにシェルモードへ切り替えるコマンド実行、VS Code 拡張、Zed・JetBrains をつなぐ ACP、Zsh プラグイン連携、マネージド／アドホック両方式の MCP サポートを備える。kimi-code への移行時には設定が自動で引き継がれる。主体は Python、ライセンスは Apache 2.0。同じ名前・同じ狙いの CLI が言語違いで並んでトレンド入りしたこと自体が、Moonshot がこの領域に本腰を入れている証左だ。

  • MoonshotAI/kimi-cli

1jehuang/jcode — 性能とスキル上限を上げにいく Rust 製 harness（週間 +2,586★ / 累計 1.1万★）

「スキルの上限を引き上げる」ことを掲げるコーディングエージェント harness。マルチセッション運用・無限のカスタマイズ性・性能を主眼に据え、Claude Code・Cursor Agent・GitHub Copilot CLI といった競合よりメモリ消費が少なく起動が速いことを前面に出す。単一セッションで 27.8MB、10 並列でも約 117MB という省メモリ、初回描画までの時間が競合比 6〜245 倍速いといった数値を並べる。

意味ベクトル埋め込みでメモリを自動検索し、背景処理で記憶を抽出・統合する記憶システム、共有リポジトリ上で複数エージェントが同時作業し衝突を自動検知・解決する「Swarm」、jcode 自身をエージェントが書き換える自己改変・リロード基盤、Claude・OpenAI・Copilot・Gemini・Azure・Ollama・LM Studio までの幅広いプロバイダ対応、Firefox 連携のブラウザ自動化を備える。主体は Rust、ライセンスは MIT。性能を売りにした Rust 製 harness が、モデルベンダー製 CLI と別の切り口でトレンドに食い込んだ。

  • 1jehuang/jcode

earendil-works/pi — LLM API からループ・TUI・CLI までを束ねるツールキット（週間 +4,495★ / 累計 7.7万★）

統一 LLM API・エージェントループ・TUI・コーディングエージェント CLI を一体で提供する AI エージェントツールキット。OpenAI・Anthropic・Google など複数プロバイダに対応し、自己拡張できるコーディングエージェントを核とする。モノレポは、多プロバイダを一枚の API に束ねる pi-ai、ツール呼び出しと状態管理を司るランタイム pi-agent-core、対話的な CLI エージェント pi-coding-agent の 3 パッケージからなり、差分レンダリングを持つ端末 UI ライブラリ pi-tui も同梱する。

依存の厳密なピン留めと整合性検証によるサプライチェーン安全性、権限を絞りたい場合のコンテナ隔離ガイド、OSS の作業セッションを公開して実利用データからエージェントを鍛える仕組みなどを特徴に挙げる。主体は TypeScript、ライセンスは MIT。単機能のエージェントではなく「エージェントを作るための土台一式」を配る路線が、今週も大きな支持を集めた。

  • earendil-works/pi

agegr/pi-web — pi コーディングエージェントのブラウザ作業場（週間 +1,050★ / 累計 2.5千★）

上記 pi コーディングエージェントのセッションファイルを読み取り、CLI 一辺倒だった操作にグラフィカルな作業場を与えるローカルの Web UI。プロジェクトごとに過去の会話を一覧・呼び出しでき、端末履歴を手で探さずに済む。

途中のメッセージから再開したり別の枝を作って探索する会話フロー、サイドバーからの git worktree 切り替え（セッションとファイルエクスプローラが追従する）、片側でプロジェクトのファイルを見ながらもう片側でソース・ドキュメント・画像・音声・PDF をプレビューする二画面構成、コンテキスト使用量・コスト・圧縮状態・システムプロンプトの可視化、モデルや認証・スキルのトグルをブラウザ上で管理する設定画面を備える。主体は TypeScript（Next.js）、ライセンスは MIT。CLI エージェントが増えるほど、こうした GUI レイヤーの需要も並んで立ち上がることを示す。

エージェントに「読むべき所」だけを渡す — 文脈の知能化

tirth8205/code-review-graph — コードベースをグラフ化し 38〜528 倍のトークン削減（週間 +6,257★ / 累計 2.6万★）

今週の実質トップ。AI コーディングツールがコードベース全体を読み込む無駄を、リポジトリの構造知識グラフで解消するローカルファーストのツール。変更の「blast radius（影響半径）」を計算し、影響を受けそうな関数・クラス・ファイルだけに文脈を絞ることで、コンテキストウィンドウの消費を大幅に減らす。実リポジトリで 38〜528 倍、中央値で約 82 倍のトークン削減を示し、fastapi では 95万トークンを 2,169 トークンまで狭めたという。

Tree-sitter でコードを AST グラフ（関数・クラスのノードと、呼び出し・継承・テストのエッジ）に落とし、ファイル変更時は 2 秒未満で差分更新する。MCP 経由で 30 の専用ツールを提供し、CLI・GitHub Actions・IDE 連携から使える。Python・JS/TS・Go・Rust・Java・C/C++ など 40 以上の言語に対応し、Codex・Claude Code・Cursor・Windsurf・Zed・Copilot などを 1 コマンドで自動設定する。D3.js の力学配置グラフによる可視化、Leiden 法によるコミュニティ検出、リスク採点付きの PR レビュー用 GitHub Action も持つ。主体は Python、ライセンスは MIT。エージェント本体の競争と並んで、「何を読ませるか」を賢くする層が今週いちばん伸びた。

  • tirth8205/code-review-graph

デザインをスキルで配る流れは今週も

ibelick/ui-skills — デザインエンジニア向けのスキル集（週間 +2,139★ / 累計 6.2千★）

「デザインエンジニアのためのスキル」を掲げる CLI ベースのツール。実証済みの UI 実装パターンとベストプラクティスをカテゴリ別に整理し、コマンドラインから発見・取得できるようにする。npx ui-skills start で開始し、カテゴリを辿ってスキル種別を一覧し、baseline-ui のような具体的な実装を引き出す、という流れを取る。

モーション表現などのカテゴリに分類された、そのまま使えるパターンを中心に据え、デザイナーやフロントエンドエンジニアが実装の手本を素早く参照して開発を加速し、プロジェクト間で一貫性を保つことを狙う。主体は JavaScript/TypeScript、ライセンスは MIT。ホストは ui-skills.com。先週の hallmark・impeccable が「AI のデザイン初期値を矯正する」検査型だったのに対し、こちらは「使えるパターンを配る」カタログ型で、デザイン×スキルの潮流に別の角度から加わった。

  • ibelick/ui-skills

先週の当欄トップだった、AI くさい見た目にならない Web UI を生成するデザインスキル。57 個の slop テストゲートと自己批評で、LLM が陥りがちなデザインの初期値を機械的に潰す。今週も週間 +5,797★ とランキング上位に残り、ui-skills の新規参入と合わせて、デザインをスキルとして配る動きが一過性でないことを裏づけた。導入は npx skills add nutlope/hallmark。主体は CSS/HTML、ライセンスは MIT。

エージェントの外側で伸びたもの

koala73/worldmonitor — 今週最大の伸び、AI で束ねる地政学インテリジェンス盤（週間 +9,054★ / 累計 7.3万★）

今週いちばんスターを集めたリポジトリ。世界中のデータを一枚のダッシュボードに集約するリアルタイム地政学インテリジェンス基盤で、AI によるニュース統合とインフラ監視を組み合わせる。15 カテゴリにまたがる 500 以上の厳選ニュースフィードを AI で処理し、3D グローブと平面 WebGL の二系統の地図で可視化する。

軍事・経済・災害のシグナルを跨いで相関させるクロスストリーム分析、主要 31 カ国のストレスを採点する Country Instability Index、29 の証券取引所や商品・暗号資産をカバーする多資産の金融レーダー、単一コードベースから world／tech／finance など 6 種のサイトを派生させる構成、Ollama によるローカル AI（API キー不要）、RTL 対応 25 言語の多言語化を備える。Web だけでなく MCP サーバー・REST API・CLI・各言語 SDK（Python/Ruby/Go）からも使える。主体は TypeScript（Three.js／deck.gl／Tauri 2）、ライセンスは AGPL-3.0。コーディング以外の領域でも、情報を束ねてエージェント連携まで用意する設計が大きな支持につながった。

  • koala73/worldmonitor

Robbyant/lingbot-map — ストリーミング映像から 3D 復元する基盤モデル（週間 +4,435★ / 累計 1.5万★）

AI エージェント一色のトレンドの中で、純粋なビジョン研究として伸びたリポジトリ。ストリーミング映像から場面を復元するフィードフォワード型の 3D 基盤モデルで、座標のグラウンディング・密な幾何手がかり・長距離のドリフト補正を単一のストリーミング枠組みに統合する「Geometric Context Transformer」を提案する。

反復最適化を要さないフィードフォワード構成と、メモリ効率の良いページド KV キャッシュ注意機構により、518×378 解像度・1 万フレーム超の系列でも約 20FPS の推論を実現するという。長尺の 3D 動画復元、リアルタイムのストリーミング地図化、屋内外シーンの取得、空マスキングを伴う動的シーン処理、空撮やウォークスルーを想定用途に挙げ、ブラウザビューアでの対話的可視化やオフラインレンダリングも備える。主体は Python、ライセンスは Apache 2.0。LLM 以外の基盤モデルもトレンドに食い込むことを示した一件だ。

  • Robbyant/lingbot-map

kvcache-ai/ktransformers — CPU/GPU 混在で巨大 MoE を回す推論・微調整基盤（週間 +1,507★ / 累計 1.9万★）

清華大学 MADSys Lab 発の、異種混合ハードウェアで LLM 推論・微調整を最適化する柔軟なフレームワーク。CPU と GPU に計算を分散し、巨大モデルを手元のハードで現実的に走らせることを狙う。AMX/AVX による CPU カーネル最適化と MoE 最適化、INT4/INT8/FP8 量子化に対応し、DeepSeek-R1（FP8）を 8×L20 GPU で約 88 トークン/秒の出力速度で回すといった数値を示す。

LLaMA-Factory と統合した SFT で DeepSeek-V3/R1 級の超巨大 MoE を少ないメモリで微調整でき、4×RTX 4090 で 3.7 iter/秒に達するという。NUMA 対応のメモリ管理、SGLang との本番連携、DeepSeek・Qwen・GLM・Kimi など新しめのアーキテクチャへの Day0 対応を備える。主体は Python、ライセンスは Apache 2.0。派手なエージェント層の下で、「巨大モデルをどう安く動かすか」の推論最適化も着実に注目を集めている。
