# 最前線

紙名: `最前線`
補助ラベル: `過去2週間`

## 調査依頼

最前線のエンジニア（フロンティアラボの中の人、著名 OSS 実践者、プロダクトを実際に出している個人）が、プロダクト開発や AI 周りについて **何を考え、どういう戦略を取り、どんなプラクティスを実験しているか** を深掘りしてまとめる。

本紙の目的は 2 つ。

1. 最前線の実践の中身を具体的に知り、自分の現在地をキャリブレーションする
2. 世の中で喧伝されている言説（誇張・嘘・浅い理解）を、最前線の実態と突き合わせて較正する

「何が起きたか」を並べる速報紙ではない。「最前線の人が何を考えているか」を読む考察紙。

逆張りに価値を置く紙ではない。声の大きい多数派のリテラシー不足が誤りを増産している状況に対して、判断基準を「実態と合っているか」に置くだけで、賛成か反対か・多数派か少数派かは重要度の判断に使わない。

## 情報源

### x（発見チャネル）

`x-list-watch` スキルで以下の list の過去 14 日分を取得する。

- 最前線 list（**未作成。作成後にここへ list ID を記入する**）: 実践者個人中心の専用 list
- 専用 list が未設定の間は LLM list (`2052860422592799160`) で代用する

x は「いま何が語られているか」の**発見チャネル**として使う。投稿単体を出典にせず、言及されている一次ソース（動画・ブログ記事・リポジトリ・トーク）まで必ず辿る。

### Web 検索・定点観測

以下を毎号確認し、加えて x で見つけた話題を深掘りする。

- Anthropic engineering blog / claude.com/blog、OpenAI などフロンティアラボの技術発信
- simonwillison.net（検証・キャリブレーション系の定点）
- 著名実践者の個人ブログ（steipete.me など、x で辿れた本人発信）
- YouTube のカンファレンストーク・本人解説動画（AI Engineer、Sequoia、ラボ公式など）
- Hacker News で議論になっている実践者の長文

## 視点とフィルタ

### 拾うもの

- 実践者**本人**による詳細な解説（30〜40 分級の動画、長文ブログ、カンファレンストーク）
- 実験中のプラクティスの**具体的な中身**（ループ構成、停止条件、検証の仕方、失敗パターンといった詳細のレベルまで語られているもの）
- 広く喧伝されている手法を、実践に基づいて検証したレポート。結論が肯定でも否定でも、実態が具体的に語られているものを拾う（失敗レポートは公開されにくいぶん希少なので、見つけたら優先的に拾う）
- 最前線の人の戦略的な思考（何に張っていて、何を捨てているか）

### 発信者の条件

実際に作って運用している人に限る。ラボの中の人、著名 OSS 実践者、プロダクトを出している個人エンジニアなど。まとめ屋・インフルエンサーによる再加工記事は出典にしない（発見の手がかりに使うのは可）。

### 対象外（他紙に譲る）

- プロダクトのリリース・機能追加そのもの → product-weekly
- 業界マクロの動向（投資・提携・規制・モデルリリース） → llm-weekly
- 研究論文単体 → llm-weekly（ただし実践者の議論と結びついていれば本紙で扱ってよい）

## 深掘りの義務

- 各記事は一次ソースに実際にあたって書く。「〜が話題になっている」で止めない
- 動画は transcript・本人の関連ポスト・詳細な解説記事で中身を具体的に把握する。中身に触れられない場合は深掘りキュー行きにし、内容を推測で書かない
- 各記事に次の 3 点を含める: (1) 誰が何を主張しているか (2) その具体的な中身 (3) 通説とのギャップ、または自分の実践への含意

## 出力構造

- **まとめ**: いま最前線がどこにいて、世間の言説とどこがズレているかを 3〜5 行で。キャリブレーションの要約として書く
- **今週の最前線**: 深掘り記事を 2〜4 本。件数より 1 本の深さを優先する
- **検証: 喧伝と実態**: 世の中で言われていること（claim）と最前線の実態を対置する。該当する素材がある号のみ。実態が言説を裏づけているならそのまま書く。対置の目的は言説の否定ではなく較正
- **深掘りキュー**: 時間を投資して全部見る・読む価値がある動画・長文を 1〜3 本。所要時間の目安と「なぜその時間を払う価値があるか」を一言添える

## 編集方針

- 鮮度より発見を優先する。公開が 2 週間より古くても、いま最前線で語られているなら扱ってよい。ただし前号で扱ったものは、進展や新しい角度がない限り再掲しない
- 重要度は「公式の派手さ」でも「バズの大きさ」でもなく、「実践の中身がどれだけ具体的に語られているか」で判断する
- 深掘りに値する素材が 2 本に満たない場合は、その号の発行を見送る（無理に薄い素材で埋めない）
